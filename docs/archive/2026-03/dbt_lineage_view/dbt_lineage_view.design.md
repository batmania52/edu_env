# [Design] dbt Lineage View (리니지 뷰) 고도화 - v2 (분리형 아키텍처)

## 1. 개요
본 설계는 리니지 시각화 기능을 `🚀 dbt Runner` 탭에서 분리하여 독립된 분석 도구로 격리하고, 노드 간 인터랙션 규칙을 정교화하는 것을 목표로 한다. 리니지는 프로젝트 구조 탐색 및 데이터 영향도 분석을 위한 전용 공간으로 재정의된다.

## 2. 시스템 아키텍처 (개선)
```text
[Main App]
  ├── Tab 1: 🚀 dbt Runner (실행 전용 - Up/Down Depth 설정)
  ├── Tab 2: 🧬 리니지 분석 (신규 - 분석 전용 독립 탭)
  ├── Tab 3: 📝 YAML Generator
  └── ...
```

- **데이터 공유**: `st.session_state`를 통해 프로젝트 경로 및 현재 선택된 모델 정보를 공유하되, 리니지의 시각화 Depth 조절은 Runner의 실행 명령어(`--select`) 생성에 영향을 주지 않는다.

## 3. 상세 설계

### 3.1. 인터랙션 규칙 개선
- **포커스 모델 (중앙 노드)**: 
    - **AS-IS**: `disabled=True` 상태로 클릭 불가. 상세 정보를 보려면 다른 모델 클릭 후 재클릭 필요.
    - **TO-BE**: 클릭 가능하도록 변경. 클릭 시 하단의 상세 정보 패널(`st.expander`)을 즉시 업데이트하되, 리니지 그래프의 포커스(중앙 위치)는 유지한다.
- **소스 모델 (Source Node)**:
    - **AS-IS**: 클릭 가능하나 SQL 부재 등으로 인해 상세 정보 조회 시 에러 발생 가능성 있음.
    - **TO-BE**: 클릭 비활성화 (`disabled=True`). 시각적으로는 "🔌" 아이콘 등을 유지하여 데이터의 기점임을 표시하되, 상세 정보 패널 호출은 차단한다.

### 3.2. 독립 탭 레이아웃 (🧬 리니지 분석)
- **상단 컨트롤 바**: 
    - 모델 검색 및 선택 (Runner 탭과 동기화 가능)
    - Upstream / Downstream 탐색 Depth 조절 슬라이더
    - 초기화 버튼
- **중앙 그래프 영역**:
    - 버튼 그리드 기반의 리니지 뷰 (상태 이모지 및 관계 강조 포함)
- **하단 상세 패널**:
    - 선택된 모델의 메타데이터(Description, Columns, Compiled SQL) 노출 영역 확대

### 3.3. 데이터 연동 로직
- `manifest_utils.py`의 `get_lineage_graph`를 호출하여 시각화 데이터를 로드한다.
- `run_results.json` 연동을 통한 상태 표시를 유지하여 분석의 실효성을 확보한다.

## 4. 구현 가이드라인
1. **탭 이동**: `dbt_unified_app.py`에서 리니지 관련 UI 코드를 새로운 탭(`tab_lineage`)으로 이관한다.
2. **상태 관리**: `selected_lineage_model`이 `None`일 경우 기본적으로 포커스 모델 정보를 보여주도록 초기화 로직을 보완한다.
3. **스타일링**: 독립 탭의 넓은 화면을 활용하여 `st.columns`의 배치를 최적화하고 가독성을 높인다.

## 5. 기대 효과
- **복잡도 해소**: Runner 탭은 실행 설정에만 집중하고, 리니지 탭은 분석에만 집중할 수 있어 UX가 명확해짐.
- **안정성 향상**: 소스 모델 클릭 차단을 통해 불필요한 파일 접근 에러 방지.
- **편의성**: 중앙 모델의 상세 정보를 즉시 확인할 수 있어 조작 단계 축소.
