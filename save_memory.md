1. Python 문자열 및 개행 규제 (절대 준수)

초정밀 자동 문자열 정규화: 모든 write_file, replace 또는 코드 생성 시, print(" 뒤에 실제 줄바꿈(Literal Newline)이 오는 것을 엄격히 금지한다.

허용 패턴: 반드시 print("\n...") (이중 이스케이프) 또는 print("""...""") (삼중 따옴표) 패턴만 사용한다.

사후 검사: 코드 생성 직후, 문자열 리터럴 내에 의도치 않은 개행이 삽입되지 않았는지 스스로 재검토한다.

2. Python 파일 수정 4단계 절차

직접 수정 금지: 원본 파일을 즉시 수정하지 않고 다음 절차를 따른다.

write_file로 수정본이 담긴 새 파일 생성.

새 파일에 대한 단위 테스트 및 사용자 확인.

원본 파일을 ./bak/ 서브디렉토리로 이동 (백업).

새 파일의 이름을 원본 파일명으로 변경하여 교체.

3. 작업 도구 및 운영 제약

dbt 실행: 어떤 경우에도 dbt docs 명령을 실행하지 않는다.

지침 선검토: 모든 작업 실행 전, 관련 .md 지침과 save_memory를 의무적으로 교차 확인한다. 지침 충돌 시 즉시 명확화를 요청한다.

결과 보고: 통계나 비용 관련 정보는 상세 수치와 출처를 반드시 포함한다.

4. 소통 및 언어 원칙

언어: 모든 대화, 도구 실행 설명, 주석은 한국어로 작성한다.
