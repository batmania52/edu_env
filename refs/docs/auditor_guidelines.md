### 6.2. 실행 파라미터 규칙

*   **모델명 자동 완성**: 사용자가 모델명만 언급하면 모델들에서 언급한 모델의 스키마를 찾은 후 `[스키마].[모델명]`으로 적용합니다.
*   **날짜 컬럼 추론1**: `before_sql`에 `truncate`가 있을 경우 `all`을 할당합니다.
*   **날짜 컬럼 추론2**: 컬럼명을 명시하지 않을 경우 모델 내 `before_sql`에서 사용하는 `where` 조건절의 `column`을 우선 사용합니다.
*   **날짜 컬럼 추론3**: 날짜 컬럼 추론2에 부합하지 않을 경우 모델 내 주요 일자 컬럼(예: `order_date`)을 우선 사용합니다.
*   **날짜 형식 변환**: 기간 요청 시 `YYYY-MM-DD` 형식으로 자동 변환하여 인자로 전달합니다.
*   **전수 검사1**: "전체" 혹은 "전수" 검사 요청 시 날짜 컬럼 인자에 `all`을 할당합니다.
*   **전수 검사2**: 모델 `sql` 파일에 `{{start}}`, `{{end}}`가 `sql`에서 사용되지 않을 경우 전수 검사로 간주하고 `all`을 할당합니다.

### 6.3. 결과 보고 및 대응

*   **요약 보고**: 실행 결과 중 `FINAL INTEGRITY SUMMARY` 섹션을 Markdown 표(Table)로 요약하여 출력합니다.
*   **오차 발생 시**: `🚨 DIFF` 발생 구간에 대해 하단의 `DEBUGGING QUERIES`를 함께 제시하며 원인 분석을 제안합니다.
*   **이상 징후 알림**: `⚠️ WARNING` (소스/타겟 모두 0건) 발생 시 데이터 유실 가능성을 사용자에게 즉시 알립니다.
