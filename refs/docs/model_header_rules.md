### 3.1. 모델 헤더 규칙

모든 SQL 모델 파일의 최상단에는 모델의 메타데이터와 변경 이력을 관리하기 위한 헤더를 작성합니다.

*   **주석 형식 및 공백**:
    *   컴파일된 SQL 파일 상단에 불필요한 빈 줄(Newlines)이 생성되는 것을 방지하기 위해 반드시 표준 SQL 주석(`/* */`)이 아닌 **Jinja 주석(`{#- ... -#}`)** 형식을 사용합니다.
    *   주석 시작과 끝에 하이픈(`-`)을 추가하여(`{#-`, `-#}`) 주변의 모든 공백과 줄바꿈을 제거합니다.
    *   파일의 가장 첫 번째 줄부터 시작하며, 주석 블록 종료 후 `config` 또는 `SELECT` 문 사이에는 **한 줄의 공백**만 허용합니다.

*   **표준 헤더 양식**:
    아래 코드 블록을 복사하여 모델 생성 시 최상단에 부착하고 내용을 기입합니다. (Jinja 주석 내부에서는 변수가 실행되지 않으므로 수동 작성을 원칙으로 합니다.)
    작성자명은 별도의 요청이 없으면 `hjpark`로 합니다.
    **변경 사항 요청이 있어 모델에 반영할 경우 Update History에 반영할것.**

    ```sql
    {#-
      [Model Information]
      - Name: [파일명 작성]
      - Developer: [작성자명]
      - Created At: YYYY-MM-DD
      - Description: [모델의 목적 및 핵심 비즈니스 로직 요약]
      
      [Update History]
      - YYYY-MM-DD: 최초 생성 ([작성자명])
      - YYYY-MM-DD: [변경 사항 요약] ([작성자명])
    -#}
    ```

*   **Description 필드 사용 지침**: `Model Information` 블록 내 `Description` 필드는 실제 참조하는 테이블/뷰의 `description` (주석)을 사용합니다.
*   **`select` 절 주석 제거 지침**: `select` 절 내에 `-- 첫 번째 컬럼은 select 키워드와 동일한 라인에 위치`와 같은 불필요한 주석은 포함하지 않습니다.
